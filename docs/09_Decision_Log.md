# Decision Log

Use this file to record product decisions so future AI tools and developers understand why choices were made.

## 2026-07-06: Do not recreate the PDF page by page

Decision:

The product should be rebuilt as a dashboard-first experience, not a digital copy of the old report.

Reason:

The old report contains useful data but weak client experience. Clients need clarity, insight, and next steps, not more pages.

Status:

Approved.

## 2026-07-06: Replace ratios with plain-language health cards

Decision:

Financial ratios should not be shown as the default view.

Reason:

Most clients do not understand ratio names. The default view should explain the meaning first.

Example:

- Liquidity Ratio becomes Emergency Fund
- Debt Servicing Ratio becomes Monthly Debt Commitment

Status:

Approved.

## 2026-07-06: Recommendations become a roadmap

Decision:

Recommendations should be grouped into Now, Next, and Later.

Reason:

This feels like a guided review journey rather than a sales push.

Status:

Approved.

## 2026-07-06: AI Coach is explanation layer

Decision:

AI Coach should explain and guide, not replace the adviser.

Reason:

The adviser relationship remains important. AI should make the client more prepared for discussion.

Status:

Approved.
