# Product Principles

## 1. Dashboard first

The first screen must give the client the answer before showing details.

## 2. No spreadsheet thinking

Tables are allowed only when they are secondary or expandable. The default view should be visual and simple.

## 3. One screen, one question

Every major screen must answer one clear user question.

Examples:

- Am I safe?
- Am I saving enough?
- What changed?
- What needs attention?
- What should I review next?

## 4. Plain language first

Technical terms can exist, but only after the user expands for details.

Example:

Use Emergency Fund instead of Liquidity Ratio.

Use Monthly Debt Commitment instead of Debt Servicing Ratio.

## 5. Insight before data

Do not show a number without explaining what it means.

## 6. Progress beats static status

Show progress, trend, and direction wherever possible.

## 7. Review, not hard sell

The product should create clarity and trust. Recommendations should look like a review roadmap, not a sales push.

## 8. Mobile must work

Many clients will view this on phone. Cards, collapsible sections, and short explanations are required.

## 9. Future-ready structure

The product should support future modules such as AI coach, family mode, retirement view, and scenario simulation.

## 10. Design like a premium product

Use clear typography, spacing, rounded cards, clean charts, subtle animation, and calm visual hierarchy.
