# Recommendation Engine

## Purpose

The product should not end with data. It should end with clear next steps.

## Principle

Recommendations should feel like a review roadmap, not a sales list.

## Structure

Use three time buckets:

```text
Now
Next
Later
```

Each item should include:

- Title
- Why it matters
- Priority
- Expected impact
- Estimated effort
- Related dashboard section
- Adviser discussion point

## Example layout

```text
Now
Review emergency fund target
Impact: High
Effort: Low
Reason: This affects short-term stability.

Next
Review protection gaps
Impact: High
Effort: Medium
Reason: This affects family risk planning.

Later
Review retirement assumptions
Impact: Medium
Effort: Medium
Reason: This affects long-term planning confidence.
```

## Priority model

Priority should consider:

- Urgency
- Impact
- Client goals
- Risk exposure
- Time sensitivity
- Ease of action

## UX requirement

Each recommendation card should be expandable.

Default view: human summary.

Expanded view: supporting numbers, assumptions, and related report data.

## Bad recommendation pattern

Avoid:

- Buy this plan
- Add more premium
- Upgrade now

Prefer:

- Review this gap
- Confirm this assumption
- Decide whether this risk matters
- Discuss available options
